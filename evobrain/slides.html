Time-then-Graph for EEG Seizure Detection

CHB-MIT, 22-channel bipolar, 12 s clips · 6 models 비교

Baseline: EvoBrain (Kotoge et al., NeurIPS 2025) · Target: 동등하거나 우수한 AUROC를 20×+ 빠르게

Time-then-Graph (TTG) Backbone

각 채널을 시간축에서 먼저 인코딩한 뒤 → 채널 간 그래프 추론.

공통 파이프라인

x ∈ ℝB×N×T×D  (N=22 채널, T=12 step, D=100 FFT bin)
──> per-channel Bi-Mamba  →  H ∈ ℝB×N×T×H
──> graph block (모델별로 다름)  →  Z ∈ ℝB×N×H
──> readout (max / PMA)  →  logit

왜 TTG?

  • EvoBrain은 매 timestep마다 동적 그래프를 재계산 → 데이터로더 병목 (47 ms/sample)
  • TTG는 시간축을 먼저 압축해서 그래프 연산 한 번 또는 K번으로 끝냄
  • 본 연구의 모든 신규 모델은 같은 Bi-Mamba 백본을 공유하고 그래프 블록만 달리함

EvoBrain Baseline

Kotoge et al., NeurIPS 2025 Spotlight. Dual Mamba (node + edge) + LapPE + GCN.

구조

Ht = NodeMamba(x:,:,t,:)
At[i,j] = corr(xi,t-w:t, xj,t-w:t)
Ât = top-k(At) ⊙ row-norm
Et = EdgeMamba(Ât)
Zt = GCN(Ht, Ât) + LapPE(Ât)
ŷ = MLP(max-pooln,t Zt)

특징

  • 그래프가 매 시점마다 재계산됨 (T=12)
  • Pearson 상관관계 → top-3 sparsify
  • LapPE = Laplacian eigenvector positional encoding
  • Edge 정보도 시간축으로 Mamba 처리

비용

  • Params 184K
  • Dataloader 47 ms/sample (correlation이 88% 차지)
  • 총 wallclock ~53 분 (50 epoch + 4 eval)

Light-Attn Baseline

가장 단순한 lightweight 기준선. 그래프를 한 번의 self-attention으로 대체.

구조

H = BiMamba(x)  ∈ ℝB×N×T×H
H̄ = meant(H)  ∈ ℝB×N×H
Q,K,V = Linear(H̄)
A = softmax(QK⊤/√d)
Z = A·V
ŷ = MLP(maxn(Z))

특징

  • 그래프 priors 없음 — 완전 학습 기반
  • 시간축은 평균으로 압축 → 그래프 한 번만
  • 가장 작은 모델 (88K)

예상

  • 구조적 정보 부족으로 AUROC 가장 낮을 것
  • 하지만 dataloader가 graph_type=none이라 매우 빠름

PLV (light_mamba_band_plv)

주파수 대역별 Phase-Locking Value로 신경과학적 prior를 그래프에 주입.

구조

// 5 bands: δ, θ, α, β, γ
for band b ∈ {δ,θ,α,β,γ}:
  xb = bandpass(raw, b)
  φib(t) = arg(Hilbert(xbi))
  PLVb[i,j] = |⟨ei(φᵢ−φⱼ)⟩t|
  Âb = top-k(PLVb) · row-norm
H̄ = meant(BiMamba(x))
Zb = GCN(H̄, Âb)  (5 branches)
g = softmax(MLP(stateMamba))
Z = Σb gb · Zb

특징

  • PLV ∈ [0,1]: 두 채널의 위상 동기화 정도
  • 발작 동안 특정 band의 동기화가 급격히 변함 → 신경생리학적 motivation
  • PLV는 clip 단위로 한 번 계산 (per-timestep ✗) → 빠름
  • 5개 branch를 Mamba state-gated fusion으로 결합

비용

  • Params 184K · ~2 분

Dyn-Hyper (light_dyn_hyper, C+)

채널이 여러 hyperedge에 동시 소속하는 soft membership을 시간에 따라 변하는 prototype으로 학습.

구조

Ht = BiMamba(x)[:,:,t,:]
// time-evolving prototypes
Qt = MLPquery(meann(Ht))
  Qt ∈ ℝEh×d, Eh=8
// soft membership
Mt[n,e] = σ(⟨Ht[n], Qt[e]⟩ / √d)
// 2-layer hypergraph conv
H't = HGConv(Ht, Mt) ×2
Z = PMA(meant(H't))

특징

  • 일반 GCN: 채널 ↔ 채널 1:1 엣지
  • Hypergraph: 채널 ↔ 그룹 (한 엣지에 여러 채널)
  • 발작은 다중 영역이 동시에 활성화 → hypergraph가 자연스러움
  • Prototype Qt가 매 시점마다 바뀜 (DETR object queries 패턴)
  • PMA = Set Transformer readout (learnable seed query)

비용

  • Params 295K · ~3 분

Static-Hyper (light_static_hyper, C-ab)

Dyn-Hyper의 ablation. Prototype을 고정된 학습 파라미터로 만듦.

구조

Ht = BiMamba(x)[:,:,t,:]
// fixed prototypes (learned)
Qstatic ∈ ℝEh×d (nn.Parameter)
// only membership varies in t
Mt[n,e] = σ(⟨Ht[n], Qstatic[e]⟩ / √d)
H't = HGConv(Ht, Mt) ×2
Z = PMA(meant(H't))

Dyn-Hyper vs Static-Hyper

  • Qt가 동적이어야 할 필요가 있는가? 라는 질문에 답하는 ablation
  • 채널-프로토타입 매칭 점수는 Ht를 통해 어쨌든 시간에 따라 변함
  • Prototype 자체가 시간에 따라 변하는 vs 채널 임베딩 변화로 충분한지

결과 미리보기

  • Static이 더 좋음 → prototype은 시간 무관 안정적 패턴으로 두는 것이 inductive bias상 유리
  • Params 147K · ~3 분

no-PLV (light_attn_band_gated)

PLV 모델의 ablation. 신경과학 prior를 학습 가능한 5개 attention head로 치환.

구조

H̄ = meant(BiMamba(x))
// 5 learned heads (PLV 자리)
for k ∈ {1..5}:
  Qk,Kk,Vk = Lineark(H̄)
  Âk = softmax(QkKk⊤/√d)
  Zk = Âk · Vk
g = softmax(MLP(stateMamba))
Z = Σk gk · Zk

PLV vs no-PLV

  • 같은 5-branch + state-gated fusion 구조
  • 차이는 branch가 무엇으로 정의되느냐
  • PLV → 주파수 대역의 phase 동기화 (fixed)
  • no-PLV → 5개 학습되는 attention pattern

의의

  • "PLV prior가 도움 되는가?"의 직접적 답
  • Params 225K (학습 헤드가 5개라 더 큼)

왜 EvoBrain은 그렇게 느린가?

  • Dataloader가 매 timestep(T=12)마다 N×N Pearson correlation 계산
  • Profile: dataloader 47 ms/sample (correlation이 88%)
  • 새 모델들은 graph_type=none → dataloader 0.71 ms/sample (67×)

Training Loss Curves

50 epoch, BCE loss, balanced sampler.

Baselines (EvoBrain, Light-Attn)는 점선. 새 모델들은 실선.

정리

1. Lightweight TTG가 EvoBrain을 따라잡는다

  • 같은 Bi-Mamba 백본에 그래프 블록만 단순화 → AUROC ≥ baseline, 20× 빠름
  • 병목은 모델이 아니라 per-timestep correlation 그래프 빌드였음

2. Hyperedge는 dynamic할 필요가 없다

  • Static-Hyper > Dyn-Hyper → prototype은 안정적 패턴으로 두는 게 좋음
  • 채널 임베딩 Ht의 변화만으로 시간 의존성은 충분

3. PLV prior는 도움이 안 됐다

  • no-PLV (학습된 attention) ≈ PLV (band-PLV prior)
  • 주파수 대역 prior가 충분히 강한 신호가 아닐 수 있음 — clip-level PLV의 한계?
  • 후속: time-windowed PLV (3s)로 다시 검증해볼 가치
